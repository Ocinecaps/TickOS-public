# TickOS — CMOS Battery Computer Proof of Concept

## What This Proves

When you boot this from USB/floppy, it writes a tiny program into your
CMOS battery-backed RAM. Then it executes that program using a
tick-stream interpreter driven by the RTC oscillator.

The program prints "OK" on screen. That "OK" was generated by code
living in your CMOS battery — not in RAM, not on disk.

**Reboot and boot from USB again**: you'll see "PROGRAM FOUND IN CMOS!"
in green, proving the program survived the power cycle. The CMOS battery
kept it alive.

## Build

Requires: `nasm` and `qemu-system-i386`

```bash
make          # assemble + create tsm.img
make run      # boot in QEMU
make clean    # remove artifacts
```

Or manually:
```bash
nasm -f bin stage1.asm -o stage1.bin
nasm -f bin stage2.asm -o stage2.bin
dd if=/dev/zero of=tsm.img bs=512 count=2880
dd if=stage1.bin of=tsm.img bs=512 seek=0 conv=notrunc
dd if=stage2.bin of=tsm.img bs=512 seek=1 conv=notrunc
qemu-system-i386 -drive format=raw,file=tsm.img,if=floppy -rtc base=localtime -boot a
```

Same thing, except row 2 shows in GREEN:
```

```

This is the proof. The program was written to CMOS on first boot.
It survived the reboot. The USB only provides the interpreter.

## How It Works

1. **Stage 1** (512 bytes): MBR bootloader, loads stage 2 from disk
2. **Stage 2** (2048 bytes): The tick-stream interpreter

The interpreter:
- Checks CMOS[0x13] for signature byte (0xBB = program exists)
- If not found: writes the "OK" program as bit-packed instructions
- Programs RTC for 8 Hz periodic interrupts
- On each tick: reads 1 bit from CMOS program page (0x30-0x3F)
- Accumulates bits MSB-first into 4-bit opcodes + operands
- Executes against 8-bit registers A/B and CMOS memory
- Display is at fixed screen positions (no scrolling)

## The "OK" Program in CMOS

```
Instruction   Binary Encoding    Bits
─────────────────────────────────────
LDA 0x14      0001 010100        10
OUT           1001               4
LDA 0x15      0001 010101        10
OUT           1001               4
HLT           1111               4
                          Total: 32 bits = 4 bytes

CMOS[0x30] = 0x15  (00010101)
CMOS[0x31] = 0x24  (00100100)
CMOS[0x32] = 0x55  (01010101)
CMOS[0x33] = 0x9F  (10011111)

Data:
CMOS[0x14] = 'O' (0x4F)
CMOS[0x15] = 'K' (0x4B)
```

At 8 Hz (8 ticks per second), it takes 32 ticks = 4 seconds to execute.
You watch each instruction decode in real time.
